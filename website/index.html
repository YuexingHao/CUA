The Problem

SKILL.md doesn't scale

Computer-using agents rely on hand-coded skill files that map intents to fixed UI coordinates. Three fundamental problems emerge at scale.

⚡

Brittleness

Hard-coded coordinates and selectors break on any UI change. A single CSS update disables entire skill libraries.

🔒

Inflexibility

Every new interaction pattern requires manual engineering. Agents can't discover behaviors autonomously.

📈

Scalability

Enterprise deployments need hundreds of skills per app. Maintenance costs grow exponentially with UI updates.

Key Finding

Supervised fine-tuning succeeds. RL catastrophically fails.

Same base model (Qwen3-8B). Same skill prediction task. Radically different outcomes depending on the learning algorithm.

Supervised LoRA

Behavioral cloning works

85.0%

LoRA fine-tuning on 1,275 multi-turn conversations with reasoning traces achieves 85% skill prediction on enterprise workflows — a 4.6x gain over zero-shot Qwen3-8B and 2.8x over Llama-70B (9x larger).

On Mind2Web, task completion jumps from 9.5% to 35.5%, with 76% cross-domain retention.

GRPO Reinforcement Learning

Reward hacking destroys generalization

4.4%

GRPO training converges in reward (0.45 → 1.37) but degrades evaluation across all benchmarks. WebArena collapses from 55.8% to 4.4% — a 92% relative drop.

Root causes: reward saturation (near-zero variance eliminates relative ranking signal), distributional collapse to 12 synthetic skill types, and catastrophic forgetting of general capabilities.

Why does GRPO fail? The composite reward (+1.0 correct skill, +0.3 format, +0.1 reasoning) saturates early, leaving near-zero variance across candidates. GRPO's relative ranking mechanism requires meaningful reward differences — without them, training degenerates into random optimization that destroys the base model's prior knowledge.

Takeaways

What we learned

✓

Training signal > model scale

An 8B model fine-tuned on interaction data outperforms a 70B model zero-shot by 2.8x. What you learn from matters more than parameter count.

✗

Naive RL destroys generalization

GRPO with coarse rewards on synthetic data causes distributional collapse. RL for skill composition needs KL regularization, multi-domain data, or finer-grained rewards.

🌐

Skills transfer across domains

Cross-domain evaluation retains 76% of in-domain performance, suggesting learned skill abstractions capture domain-agnostic behavioral patterns.

🔥

No single model dominates

Llama-70B leads on WebArena, OLMo-7B on BrowseComp+, LoRA-Qwen on IW. The best approach depends on domain diversity and task structure.
