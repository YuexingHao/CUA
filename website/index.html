The Problem

Why do computer-using agents need better skills?

Today's CUAs — agents that click, type, and scroll through GUIs to complete tasks — rely on manually authored skill files that break on every UI change and can't adapt to new domains.

The real-world failure mode

Consider an agent tasked with booking a flight. A SKILL.md agent has hard-coded coordinates for the "Search" button at position (450, 120). When the airline redesigns their page, the button moves — and the agent clicks empty space. It has no way to recover because it never understood what it was clicking, only where.

InteraSkill agents learn behavioral patterns from interaction data: "search" is a sequence of actions involving text input followed by a submit action, regardless of where UI elements appear. This understanding transfers across websites.

The stakes are real. Enterprise deployments require hundreds of hand-maintained skills per application. When UIs update — which they do constantly — teams spend weeks re-engineering skill files instead of building new capabilities.

Limitations of SKILL.md

Three fundamental problems with manual skills

💥

Brittleness

Hard-coded coordinates and selectors break on any UI change — layout shifts, element renaming, even font size changes. A single CSS update can disable an entire skill library.

🔒

Inflexibility

Every new interaction pattern requires a human to write a new skill definition. Agents can't discover behaviors autonomously — they're limited to what was anticipated and hard-coded.

📈

Scalability Crisis

Enterprise deployments need hundreds of skills per application. With 50+ apps and monthly UI updates, maintenance costs grow exponentially. This doesn't scale.

Method

Three-phase pipeline

Raw interaction trajectories in, reusable skill compositions out.

1

Trajectory Segmentation

Detect skill boundaries via action-space discontinuities. Partitions raw trajectories into candidate skill segments.

F1 = 0.851 (WebArena)
→
2

Contrastive Embedding

Cluster segments with Wasserstein distance, then refine with InfoNCE contrastive learning into a 16-dim skill space.

NMI: 0.531 → 0.871
→
3

LLM Skill Composition

Fine-tune Qwen3-8B with LoRA on multi-turn conversations with reasoning traces and failure recovery.

85.0% accuracy
