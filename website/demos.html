Hard Failure
Clicked ad banner instead of product card
Coordinate-based targeting hit an overlapping ad element.
click(0.35, 0.52) -> target: product_cardERROR: Hit element div.ad-banner (z-index: 999)Navigated to external ad URL---RECOVERY: go_back() -> re-scan DOMclick(0.35, 0.45) -> target: product_card [verified]Task recovered after 1 retry
What the Agent Learned

Coordinate-only targeting is unreliable when overlay elements exist.

Skill z3 updated: select_and_purchase

Added pre-click verification: check element.className matches expected target.

New pattern: ad_avoidance

Filter DOM elements with class containing "ad", "sponsor", "promo".

🔒
Environment Failure
Authentication wall mid-task
Session expired during a write action. Redirected to login page.
click(comment_box) -> type("Looks good")click(submit_comment)HTTP 401: UnauthorizedURL changed: /issues/42 -> /login?redirect=/issues/42---RECOVERY: Detected login pagetype(username) -> type(password) -> click(login)Redirected back -> retry comment
What the Agent Learned

Write actions can trigger auth walls.

New skill: auth_recovery (z7)

Detect login redirect. Execute credential flow, return to original URL.

New pattern: pre_auth_check

Before write actions, verify auth state by checking for user avatar.

LLM Limitation
Stale DOM reference after dynamic content load
Accessibility tree captured before AJAX content loaded.
observe(accessibility_tree) -> 12 elementsclick(element[7]) -> "Add to Cart"Element stale: DOM mutated (lazy load added 8 elements)---RECOVERY: Re-observe DOM after 500ms waitobserve() -> 20 elements -> click(element[15])Correct element targeted after re-observation
What the Agent Learned

Dynamic pages need DOM stabilization before action.

Pattern: wait_for_stable_dom

Wait until no DOM mutations for 300ms before capturing observation.

Data Loss
Only processed first page of paginated results
Returned incorrect answer because it missed 136 items on subsequent pages.
navigate(orders_page)Showing 20 of 156 ordersextract(visible_orders) -> 20 itemsWRONG: Returned cheapest from page 1 only---After learning: Now detects paginationwhile(has_next): click(next) -> collect(items)All 156 orders collected across 8 pages
What the Agent Learned

Pagination is invisible to single-page reasoning.

New skill: paginated_collect (z8)

Detect pagination indicators. Loop through all pages, collecting items.

User Input Failure
Submitted form with missing required fields
Missed a required zip code field below the fold.
type(address, "123 Main St")type(city, "Cambridge")click(submit_button)VALIDATION: "Zip code is required"---RECOVERY: Parse error -> find empty required fieldscroll(down) -> type(zip, "02139") -> click(submit)Form submitted successfully

Forms may have required fields below the fold.

Skill updated: form_completion (z9)

Before submit: scroll through entire form, identify all required fields.

Orchestration Failure
Cross-application state lost during app switch
Clipboard context lost when switching to the document editor.
[Excel] select(A1:D10) -> copy()[Switch to Word] click(document_body)[Word] paste()WRONG: Pasted stale data from previous clipboard---After learning: Use explicit data passing[Excel] export(A1:D10) -> temp_data.json[Word] import(temp_data.json) -> insert_table()Data transferred reliably via file intermediary
What the Agent Learned

Clipboard is unreliable across application switches.

New skill: cross_app_data_transfer (z10)

Export source data to structured intermediate format, then import into target.

Pattern: explicit_state_passing

Never rely on implicit OS state across app boundaries.
