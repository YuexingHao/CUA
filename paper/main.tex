\begin{abstract}
Traditional computer-using agents rely on explicitly defined skill libraries, often encoded as \texttt{SKILL.md} files mapping symbolic actions to fixed UI coordinates. Such approaches are brittle under interface changes and fail to generalize across environments. We propose an end-to-end hierarchical framework in which agents autonomously discover, encode, and compose skills from multimodal observations. Our method leverages a learned skill embedding space, mutual information maximization, and cross-modal alignment to produce reusable and semantically coherent behaviors without manual specification. Empirical and theoretical analysis suggests that learned skills improve robustness, generalization, and compositionality in complex user interface environments.
\end{abstract}

\section{Introduction}

Computer-using agents have emerged as a promising paradigm for automating complex digital workflows, spanning web navigation \cite{yao2022webshop,deng2023mind2web} to enterprise-scale task orchestration such as form filling, document management, and business reporting. These systems operate over rich graphical user interfaces (GUIs) and must execute sequences of primitive actions, including clicking, typing, and scrolling.

Despite recent progress, most state-of-the-art systems rely on manually engineered skill specifications, often materialized as configuration files (e.g., \texttt{SKILL.md}). These artifacts hardcode mappings between semantic intents and low-level execution details such as screen coordinates and UI elements. For example, a typical specification may define a \texttt{click\_submit\_button} skill as a fixed coordinate-based action.

While effective in controlled settings, this paradigm introduces three critical limitations. First, \textit{brittleness}: even minor UI changes, such as layout shifts, can invalidate predefined skills, leading to immediate failure and poor cross-domain generalization. Second, \textit{inflexibility}: extending capabilities to new interaction patterns requires manual engineering, preventing agents from autonomously discovering new behaviors. Third, \textit{lack of scalability}: large-scale systems require hundreds of manually defined skills per domain, resulting in prohibitive maintenance costs that grow with both the number of environments and their rate of change.


Formally, such agents operate under a policy:
\begin{equation}
\pi(a \mid s), \quad s \in \mathcal{S}, \quad a \in \mathcal{A}_{\text{concrete}},
\end{equation}
where $\mathcal{A}_{\text{concrete}}$ consists of fixed, predefined actions (e.g., $(x,y,\text{click})$). These mappings are typically specified in external documentation (e.g., \texttt{SKILL.md}), making them fragile to UI changes and difficult to scale.

We challenge the necessity of explicit skill specification and propose an alternative paradigm: agents should \emph{learn skills automatically} from raw interaction data without relying on manually defined skill libraries. 

Formally, we hypothesize that: \textbf{H: An agent trained to discover skills from interaction trajectories can match or exceed the performance of systems that rely on manually engineered skill specifications, while exhibiting superior generalization to unseen environments.} This hypothesis reframes the role of skills as \emph{emergent structures} rather than predefined abstractions. We posit that (i) natural clusters exist within trajectory space corresponding to meaningful behaviors, (ii) these clusters can be discovered through unsupervised learning, and (iii) the resulting representations are both interpretable and composable across tasks and domains.

We introduce a framework for unsupervised skill discovery in multimodal environments, consisting of three phases. First, \emph{skill discovery} segments interaction trajectories by identifying discontinuities in action space and clustering similar behavior segments. Second, \emph{skill embedding} learns continuous representations of these behaviors by maximizing mutual information between latent skills and trajectories, grounded in both visual and language modalities. Third, \emph{skill composition} learns hierarchical policies that combine discovered skills to solve complex tasks.

A key distinction of our approach is that both the action space and the skill abstractions are learned directly from data. Unlike prior methods that assume predefined primitives such as \{click, type, scroll\}, we operate over a continuous action representation that improves robustness to environmental variation and enables richer behavior modeling.

We evaluate our method on large-scale benchmarks, including WebShop and Mind2Web, as well as a new enterprise-oriented benchmark of GUI workflows. Across all settings, our learned-skill agent demonstrates strong generalization to unseen environments, significantly outperforming SKILL.md-based baselines in cross-domain settings. 

Notably, our approach achieves substantially higher success rates on novel domains while maintaining comparable execution efficiency. Furthermore, the learned skills are interpretable: human evaluators consistently assign semantic meaning to the discovered behaviors without access to task-specific annotations.

\section{Related Work}

\subsection{GUI Automation and the SKILL.md Paradigm}

Modern computer-using agents rely on manually specified skill libraries---often implemented as configuration files encoding mappings from semantic actions to concrete executions. This paradigm, which we term \texttt{SKILL.md}, is ubiquitous in practice yet rarely formalized in academic literature. Instead, prior work refers to it indirectly as ``action space specification,'' ``predefined primitives,'' or ``hardcoded behavior libraries.''

Early influential work including WebShop \cite{yao2022webshop} and Mind2Web \cite{deng2023mind2web} demonstrates the effectiveness of manually designed action spaces (click, type, scroll) with structured UI representations, while WebArena \cite{zhou2024webarena} and VisualWebArena \cite{koh2024visualwebarena} provide realistic evaluation environments with 812 and 910 tasks respectively. However, all reveal critical limitations: poor cross-domain generalization and inability to discover new interaction patterns.

Recent systems have pushed performance on these benchmarks substantially. OpAgent \cite{guo2026opagent} achieves 71.6\% on WebArena using a multi-agent architecture (Planner, Grounder, Reflector, Summarizer), while OpenCUA \cite{wang2025opencua} provides large-scale trajectory data (22.6K trajectories across 200+ apps) and UltraCUA \cite{yang2025ultracua} introduces hybrid GUI-plus-API actions. OSWorld \cite{xie2024osworld} extends evaluation to full desktop environments across three operating systems. Despite this progress, all these systems continue assuming predefined action primitives.

By explicitly naming and formalizing the \texttt{SKILL.md} paradigm, we expose a fundamental constraint: fixed skill libraries impose artificial dimensionality limits on agent behavior and require constant re-engineering as environments change. Our approach eliminates this requirement by jointly learning primitive actions and higher-level abstractions directly from interaction trajectory data.

\subsection{Skill Discovery and Workflow Learning for Agents}

Several recent works address the rigidity of hard-coded skills from different angles. Agent Workflow Memory (AWM) \cite{wang2024awm} induces reusable workflow routines from agent trajectories, improving WebArena success by 51.1\% relative, with both offline and online variants. SkillWeaver \cite{zheng2025skillweaver} enables agents to autonomously discover website functionalities, practice them, and distill experiences into reusable Python API skills, achieving 31.8\% improvement on WebArena and enabling cross-agent transfer. AutoManual \cite{chen2024automanual} constructs Markdown instruction manuals through agent exploration, achieving 65.1\% on WebArena's Reddit subset. Voyager \cite{wang2023voyager} demonstrates lifelong skill library accumulation in Minecraft via LLM-driven code synthesis. AgentTrek \cite{xu2025agenttrek} synthesizes training trajectories from web tutorials at scale, while WebRL \cite{qi2025webrl} and DigiRL \cite{bai2024digirl} apply reinforcement learning to improve agent policies through online interaction.

Closest to our approach, ICAL \cite{sarch2024ical} distills sub-optimal demonstrations and human feedback into cognitive abstractions, improving VisualWebArena from 14.3\% to 22.7\%. LearnAct \cite{liu2025learnact} shows that even a single demonstration improves Gemini-1.5-Pro from 19.3\% to 51.7\% on mobile GUI tasks---a 198.9\% relative gain that validates the power of interaction-derived learning signals.

However, a critical gap remains. As summarized in Table~\ref{tab:gap}, no existing work combines \emph{all four} desiderata: (1)~online skill discovery, (2)~learning from agent interaction trajectories rather than manual engineering or self-exploration, (3)~producing reusable and transferable skill definitions, and (4)~continuous improvement over time. AWM learns workflows but from LLM-based induction rather than grounded interaction patterns. SkillWeaver discovers skills autonomously but not from user-facing interaction traces. ICAL incorporates feedback but produces per-instance memories rather than transferable skill abstractions. Our framework bridges this gap.

\subsection{Unsupervised Skill Discovery and Hierarchical Abstraction}

The problem of learning reusable behaviors has been extensively studied in reinforcement learning through the options framework \cite{sutton1999between}, the option-critic architecture \cite{bacon2017option}, and unsupervised discovery methods. DIAYN \cite{eysenbach2018diversity} discovers diverse skills by maximizing mutual information $I(z; s)$ between skills and states. DADS \cite{sharma2020dads} extends this to dynamics-aware discovery by maximizing $I(z; s')$ conditioned on current state. CIC \cite{laskin2022cic} uses contrastive learning (InfoNCE \cite{oord2018cpc}) to maximize mutual information between skills and state transitions without an explicit discriminator, scaling to high-dimensional observation spaces.

However, existing skill discovery methods are typically applied to low-dimensional robotic control with predefined action spaces. Our contribution is two-fold: (1)~we extend skill discovery to high-dimensional, multimodal GUI environments where perception and action are both complex, and (2)~we learn the action space itself rather than assuming it is given. Specifically, we use trajectory-level mutual information $I(z; \tau \mid s_0)$ rather than state-level $I(z; s)$, capturing temporal structure that state-based approaches miss---a distinction critical for multi-step GUI skills such as ``fill form'' or ``navigate menu.''

\subsection{Multimodal Learning and Representation Generalization}

A core insight from representation learning is that fixed, low-dimensional specifications often fail to capture intrinsic task complexity. Studies on systematic generalization \cite{hupkes2021systematic} emphasize compositional structure, while research on intrinsic dimensionality shows that learned representations automatically discover problem structure better than handcrafted ones.

Advances in vision-language models such as CLIP \cite{radford2021clip} demonstrate that contrastive pre-training over image-text pairs yields transferable visual representations grounded in natural language. CURL \cite{srinivas2020curl} shows that contrastive objectives produce sample-efficient visual features for pixel-based RL. Our work brings similar principles to embodied GUI interaction: we align visual skill representations with language descriptions via multimodal grounding, enabling semantic generalization across UI domains with different layouts but identical task semantics. This addresses both the representation bottleneck of fixed skill libraries and the sample inefficiency of learning completely from scratch.

Learning from interaction data has a rich history: DAgger \cite{ross2011dagger} addresses covariate shift in imitation learning through iterative dataset aggregation, while Humphreys et al.\ \cite{humphreys2022data} demonstrate large-scale behavioral cloning for computer control from human demonstrations. MAML \cite{finn2017model} enables rapid few-shot adaptation---relevant to adapting discovered skills to new GUI environments. Our framework builds on these foundations but differs in producing \emph{reusable, transferable skill abstractions} rather than task-specific policies or per-instance adaptations.

Our goal is to isolate the effect of skill specification and evaluate whether unsupervised skill discovery can replace manual engineering while improving generalization and robustness.

\paragraph{Data Sources and Preprocessing.}
We construct a unified dataset by combining three large-scale sources of GUI interaction trajectories: WebShop \cite{yao2022webshop}, Mind2Web \cite{deng2023mind2web}, and a new enterprise benchmark of QBR workflows. These datasets span diverse domains, including e-commerce, real-world websites, and productivity software.

Each trajectory consists of a sequence of visual observations (screenshots), language instructions, and continuous action representations. We normalize all actions into a unified format $(x, y, a, d)$, where $(x, y)$ denote spatial coordinates, $a$ denotes action type (e.g., click, type, scroll), and $d$ denotes action duration. This normalization ensures compatibility across heterogeneous datasets.

Crucially, we construct train, validation, and test splits at the \textit{domain level}, rather than at the trajectory level. That is, all trajectories from a given domain are assigned to a single split. This design choice prevents data leakage and enables a strict evaluation of cross-domain generalization.

\paragraph{Skill Discovery.}
Given a collection of trajectories, we first perform unsupervised skill discovery. Each trajectory is segmented into candidate skills by identifying discontinuities in the action space. Specifically, we compute the difference between consecutive actions and detect boundaries where these differences exceed a high quantile threshold, indicating a transition between distinct behaviors.

Each resulting segment represents a candidate skill instance. We then cluster these segments using a distance metric based on action distribution statistics, approximating a Wasserstein-style comparison. This clustering procedure groups behaviorally similar segments into discrete skill clusters, without requiring any manual labeling or predefined skill definitions.

For each cluster, we compute a prototype skill representation by averaging the action sequences within the cluster. These prototypes serve as initial abstractions of recurring behaviors such as navigation, selection, or form interaction.

\paragraph{Skill Representation Learning.}
In the second phase, we learn continuous embeddings for the discovered skills. We train a neural encoder to map trajectory segments into a latent space, optimizing a mutual information objective between latent skill representations and observed trajectories conditioned on initial states.

This objective encourages the model to learn compact and informative representations of behavior. By leveraging contrastive learning, the model distinguishes between segments belonging to the same skill and those from different skills. This results in a structured embedding space where semantically similar behaviors are clustered together.

Importantly, this embedding process incorporates both visual observations and language inputs, enabling multimodal grounding. As a result, learned skills are not only behaviorally meaningful but also interpretable through natural language descriptions.

\paragraph{Skill Composition and Policy Learning.}
In the final phase, we train hierarchical policies that operate over the learned skill space. These policies select and compose skills to solve complex tasks, effectively treating skills as high-level actions within an options framework.

Unlike SKILL.md-based agents, which rely on manually defined multi-step procedures, our approach learns how to sequence skills directly from data. This enables the agent to generalize to novel tasks by recombining previously learned behaviors in new configurations.

\paragraph{Evaluation Protocol.}
Our evaluation spans three benchmark settings: (i) WebShop for e-commerce tasks, (ii) Mind2Web for real-world web navigation, and (iii) a newly introduced QBR Enterprise dataset for complex office workflows. This combination allows us to evaluate performance across both synthetic and real-world environments, as well as across consumer and enterprise use cases.

The QBR benchmark is particularly important as it reflects realistic business processes involving document editing, spreadsheet manipulation, and presentation generation. It serves as a stress test for evaluating whether learned skills can capture complex, multi-step workflows without manual specification.

We evaluate both learned-skill agents and SKILL.md baselines under identical conditions. The SKILL.md agent uses a fixed library of manually specified skills, each mapped to predefined action primitives and UI coordinates. In contrast, our agent learns both the action abstractions and their compositions from data.

Performance is measured in terms of task success rate, generalization to unseen domains, and execution efficiency. Generalization is specifically assessed by testing on domains that were excluded from training, ensuring a strict evaluation of cross-domain transfer.

\paragraph{Ablations and Analysis.}
We further conduct ablation studies to isolate the contribution of each component in our pipeline, including skill segmentation, embedding learning, and hierarchical composition. We also analyze the interpretability of learned skills by mapping latent representations to human-readable descriptions and measuring agreement with human annotators.

Our study design enables a controlled and scalable comparison between manual and learned skill representations. By eliminating manual skill specification and replacing it with data-driven discovery, we provide a principled evaluation of whether SKILL.md is necessary for building general-purpose computer-using agents.

\section{Problem Formulation}

We model the environment as a hierarchical Markov Decision Process (MDP):
\begin{equation}
\mathcal{M} = (\mathcal{S}, \mathcal{A}_{\text{low}}, \mathcal{A}_{\text{high}}, P, R, \gamma),
\end{equation}
where:
\begin{itemize}
    \item $\mathcal{S} \subset \mathbb{R}^{h \times w \times 3}$ is the visual observation space,
    \item $\mathcal{A}_{\text{low}} \in [0,1]^4$ is the continuous primitive action space,
    \item $\mathcal{A}_{\text{high}} = \mathcal{Z}$ is a learned latent skill space,
    \item $P$ is the transition dynamics,
    \item $R$ is the reward function,
    \item $\gamma$ is the discount factor.
\end{itemize}

To explicitly capture partial observability in UI environments, we introduce latent states $x_t \in \mathcal{X}_{\text{lat}}$ and observations $s_t \sim \Omega(\cdot \mid x_t)$. The induced belief state is:
\begin{equation}
b_t(x) = p(x_t = x \mid s_{0:t}, a_{0:t-1}),
\end{equation}
with Bayesian filtering update
\begin{equation}
b_{t+1}(x') \propto \Omega(s_{t+1} \mid x') \int_{\mathcal{X}_{\text{lat}}} P(x' \mid x, a_t) b_t(x)\, dx.
\end{equation}

The high-level controller selects a skill $z_t \sim \pi_H(\cdot \mid b_t)$ every $H$ low-level steps, while the low-level controller executes primitive actions:
\begin{equation}
a_t \sim \pi_L(\cdot \mid s_t, z_{\lfloor t/H \rfloor}).
\end{equation}

The objective is a bilevel, entropy-regularized optimization over high-level skill selection and low-level action execution. For readability, we defer the full objective, Bellman recursion, and notation details to Appendix~\ref{app:hierarchical_objective}.

\section{Hierarchical Skill Representation}

\subsection{Primitive Actions and Execution}

Unlike \texttt{SKILL.md}'s hardcoded coordinates, we represent actions in a continuous, normalized space that generalizes across UI layouts:
\begin{equation}
a_{\text{prim}} = (x, y, a_{\text{type}}, d) \in [0,1]^4,
\end{equation}
where $(x, y)$ are normalized screen coordinates (invariant to resolution), $a_{\text{type}}$ is the interaction type, and $d$ is duration. This allows the same learned distribution to work across different UIs where semantically equivalent elements appear at different locations.

We model execution noise via a stochastic policy rather than assuming deterministic point actions. The agent learns to predict action success based on visual context:
\begin{equation}
P_{\text{success}}(a \mid s) = \sigma\left( \theta_{\text{success}}^\top \phi(s) + \theta_{\text{action}}^\top a \right),
\end{equation}
where $\phi(s)$ is visual encoding. This success prediction is critical: it allows the agent to recognize when actions might fail (e.g., button has moved) and adapt accordingly. \texttt{SKILL.md} has no such mechanism—actions either work or fail silently.

% \textbf{Why this helps our goal:} Normalized, context-conditioned actions enable \emph{automatic generalization across domains} without coordinate re-specification, eliminating the need for manual SKILL.md updates when UIs change.


\subsection{Skill Embeddings: Discovering Reusable Patterns}

The core insight is that \emph{meaningful skills emerge from trajectory data without manual enumeration}. We learn a continuous skill embedding space:
\begin{equation}
z \in \mathcal{Z} = \mathbb{R}^{d_{\text{skill}}},
\end{equation}
where a small number of dimensions (e.g., 16) captures rich behavioral diversity through continuity. This contrasts sharply with discrete skill libraries that require enumerating all possible behaviors beforehand.

Each skill induces a conditional action policy:
\begin{equation}
\pi_z(a \mid s) = \mathcal{N}(\mu_z(\phi(s), z), \Sigma_z(\phi(s), z)),
\end{equation}
where the mean and variance depend on both visual state and skill embedding. This is fundamentally different from \texttt{SKILL.md}, where actions are fixed functions of task type alone. Our conditional formulation makes skills \emph{adaptive}: the same skill produces different actions depending on visual context.

Skills are discovered by maximizing trajectory-level mutual information:
\begin{equation}
\max_z \mathbb{E}_\tau \left[ \log \frac{\pi_z(\tau | s_0)}{\sum_{z'} \pi_{z'}(\tau | s_0)} \right],
\end{equation}
implemented via InfoNCE loss. This objective ensures that each skill learns to explain a distinct trajectory pattern—positive pairs (skill $z$ with its trajectories) have high likelihood, while negative pairs (skill $z'$ with irrelevant trajectories) have low likelihood. The result: skills automatically specialize and become interpretable without human intervention.

Why trajectory-level (not state-level) information? Unlike prior work maximizing $I(z; s)$, trajectory-based objectives capture \emph{temporal structure}: the sequence of actions matters, not just final outcomes. This enables discovering multi-step skills like ``fill form'' that state-based methods miss.

To ensure disentangled, interpretable skills, we apply Jacobian orthogonality regularization:
\begin{equation}
\mathcal{L}_{\text{ortho}} = \left\| J_z^\top J_z - I \right\|_F^2, \quad J_z = \frac{\partial \mu_z(s)}{\partial z}.
\end{equation}
This prevents representational collapse and ensures each skill dimension contributes meaningfully to behavior variation.

% \textbf{Why this helps our goal:} Unsupervised skill discovery eliminates manual skill enumeration entirely. Once discovered, skills automatically encode semantic meaning through their learned policies, making \texttt{SKILL.md} unnecessary.


\subsection{Skill Composition: From Primitives to Complex Behaviors}

A critical limitation of \texttt{SKILL.md} is that complex workflows must be manually engineered. Our approach learns composition automatically through a hierarchical policy using the options framework.

Each skill $z$ corresponds to a semi-Markov option with:
\begin{itemize}
    \item Policy: $\pi_z(a \mid s)$ (conditional skill policy)
    \item Termination: $\beta_z(s) \in [0,1]$ deciding when to switch skills
\end{itemize}

The option-value recursion is:
\begin{equation}
Q(s, z) = r(s, z) + \gamma \mathbb{E}_{s'} \left[ (1 - \beta_z(s')) Q(s', z) + \beta_z(s') \max_{z'} Q(s', z') \right].
\end{equation}
This enables dynamic composition: the agent learns \emph{when} to switch skills based on state feedback, not a fixed sequence. Learned termination conditions generalize across domains—e.g., ``stop filling input when field is full'' works on any form.

A macro-level policy selects which skill to execute:
\begin{equation}
\pi_{\text{macro}}(z_i \mid s_i),
\end{equation}
which sequences skills $(z_1, z_2, \ldots, z_m)$ to solve complex tasks. This hierarchical decomposition dramatically reduces learning complexity compared to end-to-end policies by breaking the problem into three levels of increasing difficulty.

Critically, compositionality enables \emph{zero-shot generalization}: if Task X on Domain A requires skills $(z_1, z_2, z_3)$ and Task Y on Domain B requires $(z_1, z_3, z_2)$, the agent solves the new task by learning only a new skill sequence, reusing all learned skill policies. This is the mechanism enabling cross-domain generalization without re-engineering.Automatic composition learning eliminates manual workflow engineering. The agent discovers task decompositions from data, enabling automatic adaptation to new domains without \texttt{SKILL.md} updates.

\section{Skill Learning and Discovery Framework}

\subsection{Mutual Information Objective and InfoNCE Loss}

The central goal is learning skills that capture distinct behavioral patterns without manual specification. We maximize mutual information between skills and trajectories:
\begin{equation}
I(z; \tau \mid s_0) = H(\tau \mid s_0) - H(\tau \mid s_0, z),
\end{equation}
which measures how much knowing a skill $z$ reduces trajectory entropy. Intuitively: high mutual information means each skill strongly predicts a specific trajectory pattern, preventing skill collapse.

We operationalize this via InfoNCE contrastive loss:
\begin{equation}
\mathcal{L}_{\text{skill}} = -\mathbb{E}_{\tau} \left[ \log \frac{\exp(f(\tau, z_\tau)/T)}{\sum_{z'} \exp(f(\tau, z')/T)} \right],
\end{equation}
where $f(\tau, z)$ measures compatibility between trajectory and skill. The positive term forces trajectories to have high likelihood under their true skill, while the denominator ensures other skills produce low likelihood. This forces each skill to specialize: skill $z_1$ learns ``click patterns,'' $z_2$ learns ``typing patterns,'' etc.

% \textbf{Why this eliminates SKILL.md:} Rather than manually assigning trajectories to skill categories, the InfoNCE loss automatically discovers which trajectory clusters correspond to meaningful skills. No human annotation needed.


\subsection{Multimodal Grounding for Cross-Domain Generalization}

Skills should be grounded in visual and semantic understanding, not just coordinates. We learn a multimodal encoder:
\begin{equation}
\phi_{\text{multi}}(v,l,a,t) = \begin{bmatrix} \phi_{\text{vision}}(v) \\ \phi_{\text{text}}(l) \\ \phi_{\text{action}}(a) \\ \phi_{\text{temporal}}(t) \end{bmatrix},
\end{equation}
combining visual state $v$ (screenshot), text $l$ (OCR labels), action history $a$, and temporal context $t$ into a shared embedding space. This allows the agent to understand \emph{semantic meaning} (``this is a save button'') independent of visual appearance.

To ensure visual and semantic representations align:
\begin{equation}
\mathcal{L}_{\text{align}} = \left\| \phi_{\text{vision}}(v_{\text{region}}) - \phi_{\text{text}}(l_{\text{label}}) \right\|_2^2,
\end{equation}
forcing visual regions (button appearance) and their semantic labels (``save'') to have similar embeddings. When the same semantic element appears on different UIs with different visual appearance, both encoders produce similar embeddings.


\subsection{Automatic Skill Discovery from Trajectories}

Given raw action trajectories, we automatically discover skill structure through three steps.

\textbf{Step 1: Trajectory Segmentation.}
We detect skill boundaries via action discontinuities:
\begin{equation}
\Delta a_t = \|a_t - a_{t-1}\|_2, \quad t \in \mathcal{B} \text{ if } \Delta a_t > \theta.
\end{equation}
This partitions each trajectory into segments, where each segment represents a candidate skill. Intuitively: when the action pattern changes significantly, we're likely switching skills (e.g., from clicking to typing).

\textbf{Step 2: Segment Representation.}
Each segment is modeled as an action distribution:
\begin{equation}
p(a \mid \text{segment}_i) = \sum_{m} \pi_m \mathcal{N}(a; \mu_m, \Sigma_m),
\end{equation}
capturing the distribution of actions within that segment. This lets us compare segments without aligning their lengths.

\textbf{Step 3: Clustering via Wasserstein Distance.}
We group similar segments into skills using:
\begin{equation}
D_W(\text{segment}_i, \text{segment}_j) = \| \mu_i - \mu_j \|_2^2 + \| \Sigma_i - \Sigma_j \|_F^2,
\end{equation}
measuring distributional similarity. Hierarchical clustering on this distance matrix automatically discovers the natural number of skills ($k$ clusters $\approx$ $k$ skills).


\subsection{Hierarchical Composition via Options Framework}

Once skills are discovered, we learn when to compose them. We model each skill as a semi-Markov option:
\begin{equation}
\mathcal{M}_{\text{options}} = (\mathcal{S}, \mathcal{Z}, P_{\text{option}}, R_{\text{option}}, \gamma),
\end{equation}
where $P_{\text{option}}(s' \mid s, z) = \int_a P(s' \mid s, a) \pi_z(a \mid s) da$ gives the multi-step transition dynamics under skill $z$.

The value of executing skill $z$ is:
\begin{equation}
Q_z(s) = \mathbb{E} \left[\sum_{t=0}^{H-1} \gamma^t r_t + \gamma^H V(s') \mid s, z\right],
\end{equation}
capturing cumulative reward over $H$ steps of skill execution. This naturally learns termination conditions: the agent stops executing $z$ when $V(s')$ (value of switching skills) exceeds continuing with $z$.

\textbf{Compositionality Guarantee.}
If skills satisfy high individual mutual information $I(z_i; \tau)$ and low correlation $I(z_i; z_j \mid \tau)$, then hierarchical policies:
\begin{equation}
\pi_{\text{macro}}(z_1, \dots, z_m \mid s_0)
\end{equation}
achieve near-optimal performance while requiring $\mathcal{O}(N/m)$ parameters compared to flat policies $\pi_{\text{flat}}(a_1, \dots, a_N \mid s_0)$. The proof follows from information bottleneck principles: if skills retain task-relevant information while compressing behavior, no performance is lost.


\subsection{Cross-Domain Generalization}

To transfer skills across domains, we define skill invariance:
\begin{equation}
\text{Inv}(z_A, z_B) = \mathbb{E}_{s_A, s_B} \left[\text{KL}(\pi_{z_A}(\cdot \mid s_A) \,\|\, \pi_{z_B}(\cdot \mid s_B))\right],
\end{equation}
measuring how similarly a skill behaves across domains. Low invariance means the skill generalizes well.

States across domains are matched semantically:
\begin{equation}
\text{Sim}(s_A, s_B) = \cos(\phi_{\text{CLIP}}(s_A), \phi_{\text{CLIP}}(s_B)) > \theta,
\end{equation}
using vision-language encoders (CLIP) to identify equivalent UI elements across visually different systems.

Given a source domain skill $z_{\text{source}}$, we adapt via few-shot learning:
\begin{equation}
z_{\text{adapted}} = z_{\text{source}} + \nabla_z \mathcal{L}_{\text{adapt}}, \quad \mathcal{L}_{\text{adapt}} = \mathbb{E}_{s} [-\log \pi_z(a^* \mid s)],
\end{equation}
where $a^*$ are expert actions on the target domain. With only 5-10 examples, this fine-tuning adapts source skills to target domain semantics.

\textbf{Transfer Guarantee:}
Under shared semantic state space assumptions:
\begin{equation}
\|\pi_{z_A} - \pi_{z_B}\|_{\text{KL}} \leq \epsilon_{\text{domain}} + \epsilon_{\text{visual}} + \epsilon_{\text{adapt}},
\end{equation}
where each error term decreases with domain similarity and adaptation data. As adaptation samples increase, $\epsilon_{\text{adapt}} \rightarrow 0$, guaranteeing eventual convergence.

% \textbf{Why this eliminates SKILL.md re-engineering:} Rather than manually creating new SKILL.md files for each domain, learned skills adapt automatically through semantic grounding and few-shot fine-tuning. Same skills work across Gmail, Outlook, and custom enterprise systems without modification.

\section{Implementation Details}

The system consists of the following components:
\begin{itemize}
    \item Vision encoder: Vision Transformer mapping $\mathcal{S} \to \mathbb{R}^{512}$
    \item Skill codebook: learnable embeddings $\mathbb{R}^{K \times d}$
    \item Skill policies: Gaussian networks $\pi_z(a \mid s)$
    \item Success predictor: binary classifier over $(s,a)$
    \item Transition model: learned dynamics for planning
\end{itemize}

Training proceeds in three phases:
\begin{enumerate}
    \item Skill discovery via clustering
    \item Representation learning via contrastive objectives
    \item Policy learning via maximum likelihood
\end{enumerate}


\section{Preliminary Experimental Results}

We validate the InteraSkill pipeline on a corpus of 500 simulated interaction trajectories spanning 12 skill types (e.g., \texttt{document\_edit}, \texttt{search\_navigate}, \texttt{send\_message}), totaling 2{,}108 ground-truth segments. All experiments were conducted on a single NVIDIA RTX A6000 GPU.

\subsection{Phase 1: Trajectory Segmentation}

We sweep the discontinuity threshold $\theta$ across percentiles of the action-difference distribution (Figure~\ref{fig:segmentation}). The best threshold ($\theta = 1.550$, 50th percentile) achieves boundary precision 0.415, recall 0.805, and F1 0.534. The high recall indicates that most true skill boundaries produce detectable action discontinuities, validating our segmentation hypothesis (H1). Precision is lower because some within-skill action variation also exceeds the threshold, suggesting that incorporating additional features (e.g., DOM state changes) could improve boundary detection.

\subsection{Phase 3: Hierarchical Skill Composition}

We evaluate three increasingly capable approaches for skill composition: (1)~an MLP baseline operating on individual skill embeddings, (2)~a Transformer decoder with causal self-attention over skill sequences, and (3)~a LoRA-fine-tuned Qwen3-8B language model trained on multi-turn user-agent conversation data including failure recovery scenarios. Results are summarized in Table~\ref{tab:composition}.

\paragraph{MLP Baseline.}
The MLP policy takes as input the current skill embedding $z_k$ concatenated with a normalized position scalar, and predicts the next skill. Trained for 150 epochs on 3{,}216 state-skill transition samples, it achieves 43\% per-step accuracy but only 5.4\% exact sequence match, as errors compound during autoregressive rollout.

\paragraph{Transformer Decoder.}
Replacing the MLP with a 2-layer causal Transformer (4 heads, $d_{\text{model}}=64$) that attends over the full sequence of skill embeddings significantly improves results. Trained for 500 epochs with teacher forcing, the Transformer achieves 7.6\% exact match and reduces normalized edit distance from 0.579 to 0.480---a 17\% relative improvement. Per-position accuracy improves uniformly (Figure~\ref{fig:comparison}), confirming that sequential context is critical for skill composition.

\paragraph{Qwen3-8B with LoRA.}
To leverage the reasoning and world knowledge of large language models, we fine-tune Qwen3-8B using QLoRA (4-bit quantization, rank-16 adapters on attention projections, 0.2\% trainable parameters). The model is trained on 1{,}275 multi-turn user-agent conversations that include: (i)~agent reasoning traces with explicit skill selection rationale, (ii)~user corrections requiring mid-workflow adaptation, and (iii)~failure recovery scenarios spanning API timeouts, permission errors, format mismatches, and cascading failures across multiple tools.

After 3 epochs of fine-tuning, the model achieves \textbf{85.0\% overall skill prediction accuracy}---a 4$\times$ improvement over the Transformer and 143\% relative gain over the MLP. Crucially, accuracy remains stable across sequence positions (80--90\% at positions 1--9), demonstrating that the LLM's contextual understanding prevents the error compounding that limits the MLP and Transformer.

\subsection{Baseline Comparison}

We compare against three external baselines to contextualize InteraSkill's skill composition performance. Table~\ref{tab:baselines} summarizes all results on both the fabricated (IW) dataset and real WebArena trajectories.

\paragraph{SKILL.md Baseline.}
A hand-coded transition table mapping each skill to its most likely successor (e.g., \texttt{search\_navigate} $\rightarrow$ \texttt{document\_edit}). This simulates the rigid, manually engineered paradigm our paper challenges. It achieves only 14.0\% mean position accuracy on IW data and 8.7\% on WebArena, confirming that static skill specifications fail to capture the diversity of real workflows.

\paragraph{AWM (Agent Workflow Memory) Baseline.}
We implement a simplified version of AWM~\cite{wang2024awm} that learns bigram transition probabilities $P(\text{skill}_{t+1} \mid \text{skill}_t)$ from training trajectories. On IW data with 12 diverse skill types, AWM achieves 33.4\% mean accuracy---comparable to our Transformer but far below the LoRA-fine-tuned LLM. On WebArena (map-only data with repetitive patterns), AWM achieves 78.8\% by learning strong self-transition priors (e.g., \texttt{search\_navigate} $\rightarrow$ \texttt{search\_navigate} at 69\%). This highlights AWM's strength on homogeneous domains but its limitation on diverse, multi-skill workflows.

\paragraph{Frequency Baseline.}
Always predicting the most common skill yields 34.9\% on IW data, establishing a lower bound. The fact that AWM barely exceeds this on diverse data suggests that simple transition statistics provide limited compositional understanding.

\section{Discussion}

The proposed framework removes the need for explicit skill specification and enables agents to autonomously discover reusable behaviors. Our experiments reveal several key findings.

\textbf{Skill discovery works.} Phase 2 achieves NMI = 0.871 and purity = 0.869 on fabricated data, demonstrating that contrastive learning recovers semantically meaningful skill clusters from trajectory data alone. On real WebArena trajectories, segmentation quality is even higher (F1 = 0.851), though embedding quality is lower due to limited domain diversity in the current WebArena sample.

\textbf{Composition requires rich context.} The progression from MLP (21\%) to Transformer (35\%) to Qwen3-8B LoRA (85\%) demonstrates that skill composition is fundamentally a contextual reasoning problem, not a pattern matching one. Static transition models (AWM, frequency baseline) plateau at $\sim$35\% on diverse data because they lack the capacity to reason about task goals, user corrections, and failure recovery. The LLM's ability to leverage conversation history, reasoning traces, and world knowledge yields a 2.4$\times$ improvement over the best non-LLM approach.

\textbf{SKILL.md is consistently worst.} Across both datasets, the fixed transition table achieves the lowest accuracy (8.7--14.0\%), validating our central thesis: hard-coded skill specifications are brittle and should be replaced by learned representations.

\textbf{Failure recovery is learnable.} By training on multi-turn conversations that include API timeouts, permission errors, format mismatches, and cascading failures, the LoRA-fine-tuned model learns not only which skill to execute but how to diagnose failures and adapt. This capability---absent from all baselines---is critical for real-world deployment where tool interactions frequently fail.

\textbf{Limitations.} Our current WebArena evaluation is limited to map navigation trajectories. The fabricated data, while diverse in skill types, lacks the visual grounding and DOM-level complexity of real web interactions. The Qwen3-8B evaluation uses text-based skill prediction rather than end-to-end task execution. Future work will integrate InteraSkill's skill composition with a full WebArena agent to measure end-to-end task success rates.

\section{Conclusion}

By integrating segmentation, clustering, and representation learning into a unified framework, we eliminate the need for manually curated skill specifications. The resulting agents exhibit emergent structure, compositional reasoning, and cross-domain generalization, suggesting a scalable path toward general-purpose computer-using systems.
